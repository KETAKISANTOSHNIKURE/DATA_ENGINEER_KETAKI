## 🔑 Quick Problem Identification

| Keywords in Problem | Solution Pattern |
|---------------------|------------------|
| "second highest", "nth", "top N" | `DENSE_RANK()`, `ROW_NUMBER()`, `LIMIT/OFFSET` |
| "duplicate", "more than once" | `GROUP BY` + `HAVING COUNT(*) > 1` |
| "never", "did not", "without" | `LEFT JOIN` + `WHERE col IS NULL` or `NOT IN` |
| "consecutive", "in a row" | Self-join or `LAG()/LEAD()` |
| "running total", "cumulative" | `SUM() OVER (ORDER BY)` |
| "per group", "each department" | `PARTITION BY` in window function |
| "combine", "merge two tables" | `LEFT JOIN` |

1. **Start with** [00-problem-identification-guide](./docs/00-problem-identification-guide.md) – learn how to recognize problem types
2. **Practice by category** – Basic → JOINs → Aggregation → Window Functions → Advanced
3. **Memorize keyword mappings** – "second highest" → `DENSE_RANK` or `LIMIT 1 OFFSET 1`
4. **Focus on patterns** – Most problems repeat the same 10–15 patterns

## 🎯 Interview Tips

- **Read carefully** – "second highest" vs "second highest (unique)" changes the approach
- **Handle NULLs** – Use `IFNULL`, `COALESCE`, or `IS NULL` checks
- **Test edge cases** – Empty table, single row, all duplicates
